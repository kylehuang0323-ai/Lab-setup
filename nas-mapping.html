⚠️ 为什么不用 persistent 映射？

因为回滚软件（Reboot Restore Rx Pro）会在重启时还原系统状态， 包括注册表中的 persistent 网络映射记录。所以我们必须用登录脚本方式， 每次登录时自动重新映射，确保 NAS 盘始终可用。

4.1 创建 NAS 映射脚本

在 PowerShell (Admin) 中执行以下命令创建脚本文件：

# 创建脚本目录 New-Item -Path "C:\Scripts" -ItemType Directory -Force # 创建 NAS 映射批处理脚本 $scriptContent = @' @echo off REM ============================================= REM I-Lab NAS Auto-Mount Script REM 每次登录时自动执行，映射 NAS 网络盘 REM ============================================= REM 等待网络就绪（开机后网络可能需要几秒初始化） timeout /t 5 /nobreak >nul REM 清除旧映射（防止冲突） net use Z: /delete /yes 2>nul net use Y: /delete /yes 2>nul REM 映射 LabData 为 Z: 盘（用户工作数据） net use Z: \\192.168.0.81\LabData /persistent:no /user:labuser YourPasswordHere REM 映射 Software 为 Y: 盘（软件仓库，只读参考） net use Y: \\192.168.0.81\Software /persistent:no /user:labuser YourPasswordHere REM 验证映射 if exist Z:\ ( echo [OK] Z: drive mounted successfully ) else ( echo [WARN] Z: drive mount failed, retrying... timeout /t 3 /nobreak >nul net use Z: \\192.168.0.81\LabData /persistent:no /user:labuser YourPasswordHere ) '@ $scriptContent | Out-File -FilePath "C:\Scripts\mount-nas.bat" -Encoding ASCII Write-Host "[OK] Script created: C:\Scripts\mount-nas.bat" -ForegroundColor Green
⚠️
安全提示：脚本中的密码是明文。对于 Lab 环境可接受。 如需更安全，可以在 NAS 端对该共享设置"允许 Guest 访问"，然后脚本中不写密码： net use Z: \\192.168.0.81\LabData /persistent:no

4.2 注册为登录自动运行

有两种方式，推荐方式 B（Task Scheduler 更可靠）：

方式 A：注册表 Run 键

# 所有用户登录时自动运行映射脚本 reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "MountNAS" /t REG_SZ /d "C:\Scripts\mount-nas.bat" /f # 验证注册表已写入 reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "MountNAS"

方式 B：Task Scheduler（推荐）

# 创建计划任务：每次用户登录时运行映射脚本 $action = New-ScheduledTaskAction -Execute "C:\Scripts\mount-nas.bat" $trigger = New-ScheduledTaskTrigger -AtLogOn $settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable $principal = New-ScheduledTaskPrincipal -GroupId "BUILTIN\Users" -RunLevel Limited Register-ScheduledTask ` -TaskName "I-Lab-Mount-NAS" ` -Action $action ` -Trigger $trigger ` -Settings $settings ` -Principal $principal ` -Description "Auto-mount NAS drives on user login" ` -Force # 验证任务已创建 Get-ScheduledTask -TaskName "I-Lab-Mount-NAS" | Format-Table TaskName, State

4.3 创建桌面数据保存提示

在公共桌面放一个醒目的提示文件，所有用户登录都能看到：

# 创建桌面提示文件（所有用户都能看到） $notice = @" ╔══════════════════════════════════════════════════════╗ ║ I-Lab 工作站使用须知 ║ ╠══════════════════════════════════════════════════════╣ ║ ║ ║ ⚠️ 重要：本机重启后 C: 盘会自动还原！ ║ ║ ║ ║ ✅ 请将所有工作文件保存到： ║ ║ Z: 盘（NAS 网络盘） ║ ║ 路径示例：Z:\你的名字\项目名\ ║ ║ ║ ║ ❌ 以下位置的文件重启后会消失： ║ ║ • 桌面 (Desktop) ║ ║ • 文档 (Documents) ║ ║ • 下载 (Downloads) ║ ║ • C: 盘上任何位置 ║ ║ ║ ║ 💡 使用建议： ║ ║ • 代码项目 clone 到 Z:\你的名字\projects\ ║ ║ • VS Code 工作区保存到 Z: 盘 ║ ║ • Y: 盘有常用软件安装包（只读） ║ ║ • 如需安装新软件，联系 Lab 管理员 ║ ║ ║ ║ 📞 遇到问题？联系 Lab 管理员 ║ ║ ║ ╚══════════════════════════════════════════════════════╝ "@ $notice | Out-File -FilePath "C:\Users\Public\Desktop\⚠️ 数据保存须知-必读.txt" -Encoding UTF8 Write-Host "[OK] Notice file created on Public Desktop" -ForegroundColor Green

4.4 创建快捷方式到 NAS

# 在公共桌面创建 Z: 盘快捷方式，方便用户一键打开 $WshShell = New-Object -ComObject WScript.Shell $shortcut = $WshShell.CreateShortcut("C:\Users\Public\Desktop\📁 NAS 工作盘 (Z).lnk") $shortcut.TargetPath = "Z:\" $shortcut.Description = "I-Lab NAS 数据盘 - 在这里保存你的工作文件" $shortcut.Save() Write-Host "[OK] NAS shortcut created on Public Desktop" -ForegroundColor Green

4.5 设置 Windows 默认保存位置（可选）

将 Windows 默认的文档/下载位置指向 NAS（对部分应用有效）：

# 注意：此设置会被回滚软件还原，但首次使用时有引导作用 # 如果确定不需要可跳过此步 # 创建 NAS 上的模板目录结构 if (Test-Path "Z:\") { New-Item -Path "Z:\Template" -ItemType Directory -Force New-Item -Path "Z:\Template\Desktop" -ItemType Directory -Force New-Item -Path "Z:\Template\Documents" -ItemType Directory -Force New-Item -Path "Z:\Template\Projects" -ItemType Directory -Force Write-Host "[OK] Template directories created on NAS" -ForegroundColor Green } else { Write-Host "[SKIP] Z: not mounted, skip NAS directory creation" -ForegroundColor Yellow }

4.6 测试验证

# 模拟测试：注销再登录，检查映射是否自动生效 # 或者手动运行脚本测试： & "C:\Scripts\mount-nas.bat" # 验证 Z: 盘可访问 if (Test-Path "Z:\") { Write-Host "[OK] Z: drive accessible" -ForegroundColor Green Get-ChildItem Z:\ | Select-Object Name, LastWriteTime } else { Write-Host "[FAIL] Z: drive not accessible!" -ForegroundColor Red } # 验证写入权限 try { "test" | Out-File "Z:\connection_test.txt" Remove-Item "Z:\connection_test.txt" Write-Host "[OK] Z: drive read/write OK" -ForegroundColor Green } catch { Write-Host "[FAIL] Z: drive write permission denied!" -ForegroundColor Red } # 验证桌面文件存在 if (Test-Path "C:\Users\Public\Desktop\⚠️ 数据保存须知-必读.txt") { Write-Host "[OK] Desktop notice file exists" -ForegroundColor Green }
  • C:\Scripts\mount-nas.bat 文件存在

  • Z: 盘可读写文件
  • Y: 盘可读取
  • 公共桌面有"数据保存须知"文件
  • 公共桌面有 NAS 快捷方式

4.7 Sysprep 阻塞包快速检查（封装前）

如果执行 Sysprep 弹出 “was not able to validate your Windows installation”，先在管理员 PowerShell 执行以下排查命令。

# 检查“已安装但未预配”的 Appx 包（常见阻塞源） $aProv=@(Get-AppxProvisionedPackage -Online).PackageName Get-AppxPackage -AllUsers | Where-Object { $aProv -notcontains $_.PackageFullName } | Select Name, PackageFullName # 查看 Sysprep 最近错误 Get-Content "C:\Windows\System32\Sysprep\Panther\setuperr.log" -Tail 80 # 若看到 LanguageExperiencePackzh-CN / 0x3cf2，可执行： Get-AppxPackage *LanguageExperiencePackzh-CN* | Remove-AppxPackage Get-AppxPackage -AllUsers *LanguageExperiencePackzh-CN* | Remove-AppxPackage -AllUsers Get-AppxProvisionedPackage -Online | Where-Object { $_.DisplayName -like "*LanguageExperiencePackzh-CN*" } | Remove-AppxProvisionedPackage -Online # 若看到 0x800F0975（reserved storage / update in use） UsoClient StartScan Restart-Computer # 处理完成后重试 Sysprep & "$env:WINDIR\System32\Sysprep\Sysprep.exe" /oobe /generalize /shutdown
