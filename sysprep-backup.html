📋 Step 5: 最终清理 & 验证（15 min）

5.1 最终验证 Checklist

在执行清理前，最后确认所有配置正确：

  • Device Manager 零感叹号（所有驱动正常）
  • 所有软件可正常打开（VS Code, Chrome, Python, Arduino IDE）
  • python --version / git --version / node --version 正常
  • NAS Z: 盘可访问，可读写文件
  • Active Backup Agent 显示已连接
  • 注销再登录后 NAS 自动映射
  • 公共桌面有"数据保存须知"文件和 NAS 快捷方式
  • 远程桌面可从其他设备连入
  • 分辨率、声音、USB 口正常
  • 时区 UTC+8、电源不休眠
⚠️
如果有任何一项未通过，先回到对应步骤修复。Sysprep 后无法修改！

5.2 清理临时文件

减小镜像体积，清除不必要的缓存：

# ===== 清理 Windows 组件存储（释放 1-3GB）===== Dism.exe /online /Cleanup-Image /StartComponentCleanup /ResetBase # ===== 清理临时文件 ===== Remove-Item -Path "$env:TEMP\*" -Recurse -Force -ErrorAction SilentlyContinue Remove-Item -Path "C:\Windows\Temp\*" -Recurse -Force -ErrorAction SilentlyContinue # ===== 清空回收站 ===== Clear-RecycleBin -Force -ErrorAction SilentlyContinue # ===== 清理 Windows Update 缓存 ===== Stop-Service -Name wuauserv -Force -ErrorAction SilentlyContinue Remove-Item -Path "C:\Windows\SoftwareDistribution\Download\*" -Recurse -Force -ErrorAction SilentlyContinue Start-Service -Name wuauserv # ===== 清理浏览器缓存 ===== Remove-Item -Path "$env:LOCALAPPDATA\Google\Chrome\User Data\Default\Cache\*" -Recurse -Force -ErrorAction SilentlyContinue Remove-Item -Path "$env:LOCALAPPDATA\Microsoft\Edge\User Data\Default\Cache\*" -Recurse -Force -ErrorAction SilentlyContinue # ===== 清理 pip 缓存 ===== pip cache purge 2>$null # ===== 清理 npm 缓存 ===== npm cache clean --force 2>$null # ===== 运行磁盘清理（GUI 方式，全选所有项目）===== # cleanmgr /d C /sageset:1 ← 第一次设置：全部勾选 # cleanmgr /d C /sagerun:1 ← 之后运行 cleanmgr /d C /verylowdisk

5.3 检查清理效果

# 查看 C: 盘使用情况 Get-PSDrive C | Select-Object @{N='Used(GB)';E={[math]::Round($_.Used/1GB,1)}}, @{N='Free(GB)';E={[math]::Round($_.Free/1GB,1)}} # 目标：系统+所有软件应在 30-50GB 左右 # 如果超过 60GB，检查是否有大文件未清理
🔒 Step 6: Sysprep 封装（10 min）

6.1 什么是 Sysprep？

作用说明
移除机器 SID确保克隆后每台机器有唯一身份，不会网络冲突
移除硬件绑定驱动会在首次启动时重新检测安装
重置激活状态Windows 会在首次启动时重新激活
进入 OOBE下次启动时显示初始设置向导（设机器名等）

6.2 检查 Sysprep 兼容性

# 某些应用（特别是 Windows Store 应用）可能阻止 Sysprep # 先检查潜在问题： # 列出可能有问题的 Appx 包 Get-AppxPackage | Where-Object {$_.IsFramework -eq $false -and $_.SignatureKind -ne "System"} | Select-Object Name, Version | Format-Table # 检查“已安装但未预配”的包（Sysprep 常见阻塞源） $aProv=@(Get-AppxProvisionedPackage -Online).PackageName Get-AppxPackage -AllUsers | Where-Object { $aProv -notcontains $_.PackageFullName } | Select Name, PackageFullName # 如果 Sysprep 失败，通常需要移除有问题的应用： # Get-AppxPackage *应用名* | Remove-AppxPackage # Get-AppxProvisionedPackage -Online | Where-Object {$_.PackageName -like "*应用名*"} | Remove-AppxProvisionedPackage -Online
💡
常见需要移除的应用：Copilot, Xbox Game Bar, Microsoft Teams (consumer)。 企业/学校版 Store 应用通常没问题。

6.3 执行 Sysprep

🚨
最后确认：
  • Step 1-5 全部完成？
  • 确认不需要再装任何软件？
  • Deep Freeze 与 ABB 策略已准备好？
  • 执行后系统会关机，不能再正常开机！

方法 A：命令行执行（推荐）

# 以管理员权限运行： C:\Windows\System32\Sysprep\sysprep.exe /generalize /oobe /shutdown # 参数说明： # /generalize → 移除硬件特定信息（SID、驱动绑定等） # /oobe → 下次启动进入初始设置界面 # /shutdown → 完成后关机（不是重启！）

方法 B：图形界面执行

# 1. 打开：C:\Windows\System32\Sysprep\sysprep.exe # 2. System Cleanup Action: 选择 "Enter System Out-of-Box Experience (OOBE)" # 3. 勾选 ☑ Generalize # 4. Shutdown Options: 选择 "Shutdown" # 5. 点击 OK

等待过程

# Sysprep 运行过程： # • 屏幕显示 "Sysprep is working on your request..." # • 持续约 2-5 分钟 # • 完成后系统自动关机 # • 屏幕变黑 = 完成

⛔ 关机后的操作

1. 不要按电源键！
2. 进入 MIX 控制台
3. 按 Step 7 固化 Deep Freeze 与 ABB 基线

6.4 Sysprep 失败处理

# 如果 Sysprep 报错，查看日志： Get-Content "C:\Windows\System32\Sysprep\Panther\setuperr.log" -Tail 30 Get-Content "C:\Windows\System32\Sysprep\Panther\setupact.log" -Tail 50 # 常见错误及解决： # # 错误 A：There are one or more Windows updates that require a reboot # 解决：先重启，再重试 Sysprep（必要时再重启一次） Restart-Computer # 错误 A-1：0x800F0975（reserved storage/update in use） # 解决：先完成更新并重启，再执行 Sysprep UsoClient StartScan Restart-Computer # 错误 B：BitLocker-Sysprep: BitLocker is on for the OS volume # 解决：必须先完成 C 盘解密，再执行 Sysprep Disable-BitLocker -MountPoint "C:" manage-bde -status C: # 等待直到显示： # Conversion Status: Fully Decrypted # Percentage Encrypted: 0% # 错误 C：Package ... was installed for a user, but not provisioned for all users # 解决：按报错包名移除当前用户、全部用户、预配包 Get-AppxPackage *包名* | Remove-AppxPackage Get-AppxPackage -AllUsers *包名* | Remove-AppxPackage -AllUsers Get-AppxProvisionedPackage -Online | Where-Object {$_.DisplayName -like "*包名*"} | Remove-AppxProvisionedPackage -Online # 本次实测示例（中文语言体验包） Get-AppxPackage *LanguageExperiencePackzh-CN* | Remove-AppxPackage Get-AppxPackage -AllUsers *LanguageExperiencePackzh-CN* | Remove-AppxPackage -AllUsers Get-AppxProvisionedPackage -Online | Where-Object {$_.DisplayName -like "*LanguageExperiencePackzh-CN*"} | Remove-AppxProvisionedPackage -Online # 该类错误常见码：0x3cf2 # # 错误：SYSPRP Failed to remove apps for the current user # 解决： # reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Appx\AppxAllUserStore\Deprovisioned\Microsoft.XXX_8wekyb3d8bbwe" /f # # 错误：Sysprep count exceeded (已运行过太多次) # 解决：重新安装 Windows（从 Step 1 重来） # 重新执行 Sysprep： & "$env:WINDIR\System32\Sysprep\Sysprep.exe" /oobe /generalize /shutdown

7.1 策略固化（MIX 控制台）

# 目标：把当前可用状态固化为可回滚基线 # 1) 策略名：WS-Standard-Frozen # 2) 冻结盘：C: # 3) 数据盘：Z:（NAS 持久化） # 4) 维护窗口：每周三 02:00-04:00（可按站点调整） # 5) 发布到 WS-Pilot，再推广 WS-Lab-Prod

# C盘写测试文件（应回滚） New-Item -ItemType Directory -Path C:\DF-Test -Force "rollback-test $(Get-Date)" | Out-File C:\DF-Test\rollback-test.txt -Encoding utf8 # Z盘写测试文件（应保留） "persist-test $(Get-Date)" | Out-File Z:\df-persist-test.txt -Encoding utf8 # 重启后检查： # 1) C:\DF-Test\rollback-test.txt 不存在（回滚生效） # 2) Z:\df-persist-test.txt 存在（持久化生效）

🖥️ Step 8: 12 台工作站镜像分配（后台直配）

8.1 是否可以后台直接分配镜像？

✅
可以。在完成 Step 5-7 的黄金镜像封装后，可在后台按“机器 → 镜像”直接分配并下发到指定工位，无需每台手动重装。
场景后台动作结果
新工位上线（WS-001 ~ WS-012）绑定默认镜像（如 Win11-Base-v1）首次开机自动部署统一基线
指定用户要特定镜像把目标机器改绑到对应镜像（如 AI-Lab-v2）下次部署窗口按新镜像重建
临时试验镜像分配到试验分组（WS-Pilot）验证通过后再推广到 WS-Lab-Prod

8.2 推荐后台分配流程（面向 12 台）

# 1) 准备镜像目录（示例命名） # Win11-Base-v1（通用） # AI-Workshop-v1（演示环境） # Dev-Tools-v1（开发工具集） # 2) 建立设备分组 # WS-Pilot: WS-001, WS-002 # WS-Lab-Prod: WS-003 ~ WS-012 # 3) 后台绑定规则 # 组默认镜像：WS-Lab-Prod -> Win11-Base-v1 # 机器覆盖规则：WS-007 -> AI-Workshop-v1（高优先级） # 4) 发布策略 # 先发 WS-Pilot（2 台）验证，再全量发到 WS-Lab-Prod # 5) 验收清单（每台） # 镜像版本正确、Deep Freeze 状态正确、Z: 持久化正常、ABB 备份成功
💡
建议规则：优先使用“分组默认镜像 + 单机覆盖镜像”。这样既能保证 12 台统一，又能满足“这台机器要特定镜像”的临时诉求。

8.3 变更控制（避免误下发）

控制项建议
镜像发布窗口固定在非上课时段（如周三 02:00-04:00）
审批机制生产组变更需二次确认（管理员 + 值班负责人）
回滚策略保留上一版镜像 2 个周期，可一键切回
命名规范镜像名包含用途与版本：AI-Workshop-v1

🔧 常见问题排查（Deep Freeze 模型）

问题原因解决
重启后 C盘文件还在 设备未进入 Frozen / 策略未命中 检查设备分组与策略下发，强制同步并重启
Z盘文件丢失 误写到 C 盘或映射异常 核对路径与映射脚本，确认写入 Z:\ 后复测
ABB 显示等待设备 Agent 会话异常或连通性问题 Agent 断开重连、重启服务、检查端口 5510/443
