2.1 打开 PowerShell（管理员模式）

右键点击开始菜单 → 选择 Terminal (Admin)，或搜索 "PowerShell" → Run as Administrator

⚠️
以下所有命令都必须在管理员 PowerShell 中运行，否则会报权限错误。

2.2 计算机名 & 区域设置

# 设置临时模板名（Sysprep 后会被清除，仅制作阶段使用） Rename-Computer -NewName "GMK-WS-TEMPLATE" -Force # 设置时区为中国标准时间 (UTC+8) Set-TimeZone -Id "China Standard Time" # 验证时区设置 Get-TimeZone # 预期输出：(UTC+08:00) Beijing, Chongqing, Hong Kong, Urumqi # 设置系统区域为中国（影响日期格式、货币等） Set-WinSystemLocale -SystemLocale zh-CN # 设置位置为中国 Set-WinHomeLocation -GeoId 45

2.3 电源策略

确保工作站不会自动休眠/关屏，影响用户操作和远程管理。

# 接通电源时永不待机（0 = 从不） powercfg /change standby-timeout-ac 0 # 禁用 hibernate（避免占用磁盘空间 + 还原冲突） powercfg /change hibernate-timeout-ac 0 powercfg /hibernate off # 显示器 60 分钟后关闭（节能但不影响运行） powercfg /change monitor-timeout-ac 60 # 硬盘永不关闭 powercfg /change disk-timeout-ac 0 # 禁用 USB 选择性挂起（防止 Arduino/传感器/开发板断连） powercfg /SETACVALUEINDEX SCHEME_CURRENT 2a737441-1930-4402-8d77-b2bebba308a3 48e6b7a6-50f5-4782-a5d4-53bb8f07e226 0 powercfg /SETACTIVE SCHEME_CURRENT

验证电源配置

# 查看当前活动电源方案的所有设置 powercfg /query SCHEME_CURRENT # 快速检查关键项 Write-Host "=== Power Settings Check ===" -ForegroundColor Cyan $plan = powercfg /query SCHEME_CURRENT if ($plan -match "Current AC Power Setting Index:\s+0x00000000") { Write-Host "[OK] Standby timeout = Never" -ForegroundColor Green }

2.4 开启远程桌面 (RDP)

管理员远程维护使用，无需每次到机器前操作。

# 开启远程桌面服务 Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name "fDenyTSConnections" -Value 0 # 防火墙放行 RDP（端口 3389） Enable-NetFirewallRule -DisplayGroup "Remote Desktop" # 设置 NLA（Network Level Authentication）- 更安全 Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp' -Name "UserAuthentication" -Value 1 # 验证 RDP 已开启 $rdp = Get-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' if ($rdp.fDenyTSConnections -eq 0) { Write-Host "[OK] Remote Desktop enabled" -ForegroundColor Green } else { Write-Host "[FAIL] Remote Desktop NOT enabled!" -ForegroundColor Red }
💡
测试方法：从另一台电脑运行 mstsc /v:GMK-WS-TEMPLATE 尝试连接。

2.5 开启 Wake on LAN (WoL)

允许管理员从 NAS 远程唤醒关机状态的工作站。

PowerShell 配置（部分网卡支持）

# 查看所有网卡 Get-NetAdapter | Format-Table Name, Status, LinkSpeed, MacAddress # 查看网卡高级属性中是否有 WoL 相关设置 Get-NetAdapterAdvancedProperty -Name "Ethernet*" | Where-Object {$_.DisplayName -like "*Wake*" -or $_.DisplayName -like "*Magic*"} | Format-Table DisplayName, DisplayValue

手动设置（必须做）

# 需在设备管理器中手动操作： # # 1. 右键开始菜单 → Device Manager # 2. 展开 Network adapters # 3. 右键 Realtek 2.5G 网卡 → Properties # 4. 切换到 Power Management 标签页 # 5. 勾选以下选项： # ☑ Allow this device to wake the computer # ☑ Only allow a magic packet to wake the computer # 6. 切换到 Advanced 标签页 # 7. 找到 "Wake on Magic Packet" → 设为 Enabled # 8. 点 OK 保存

BIOS 中确认 WoL（如未设置）

# GMK BIOS 中 WoL 设置路径： # 开机按 DEL 进入 BIOS # Advanced → ACPI Settings → Wake on LAN → Enabled # 如果没有此选项，可能在： # Advanced → Network Stack Configuration → Network Boot → Enabled # Save & Exit

2.6 其他优化设置

# 禁用 Windows Tips/建议通知（减少用户干扰） reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\CloudContent" /v "DisableWindowsConsumerFeatures" /t REG_DWORD /d 1 /f reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v "SubscribedContent-338389Enabled" /t REG_DWORD /d 0 /f # 禁用 Cortana reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\Windows Search" /v "AllowCortana" /t REG_DWORD /d 0 /f # 显示文件扩展名（对开发者重要） reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" /v "HideFileExt" /t REG_DWORD /d 0 /f # 显示隐藏文件 reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" /v "Hidden" /t REG_DWORD /d 1 /f # 设置 Windows Update 为仅通知，不自动重启 reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v "NoAutoRebootWithLoggedOnUsers" /t REG_DWORD /d 1 /f # 禁用 Windows Spotlight 锁屏广告 reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\ContentDeliveryManager" /v "RotatingLockScreenOverlayEnabled" /t REG_DWORD /d 0 /f

2.7 无密码自动登录配置

💡
设计原理：I-Lab 工作站采用"物理在场即认证"模型。
用户通过门禁/认证器进入实验室 = 已完成身份验证。
工作站无密码 + 每日自动重置 = 零门槛使用 + 无持久化风险。
安全边界由物理管控 + 网络隔离 + NAS 权限 + 每日回滚保障，而非本机密码。

配置命令（管理员权限执行）

REM === 1. 设置账户空密码 === net user Lab-Admin "" REM === 2. 允许空密码本地登录（默认被安全策略阻止） === reg add "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v LimitBlankPasswordUse /t REG_DWORD /d 0 /f REM === 3. 设置开机自动登录 === reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon /t REG_SZ /d "1" /f reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUserName /t REG_SZ /d "Lab-Admin" /f reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultPassword /t REG_SZ /d "" /f REM === 4. 禁用锁屏 === reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\Personalization" /v NoLockScreen /t REG_DWORD /d 1 /f REM === 5. 禁用 Ctrl+Alt+Del 登录要求 === reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v DisableCAD /t REG_DWORD /d 1 /f REM === 6. 禁用 Windows Hello 提示与自动重启登录 === reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v DisableAutomaticRestartSignOn /t REG_DWORD /d 1 /f

各命令说明

#命令作用为什么需要
1net user Lab-Admin ""清除账户密码确保账户无需输入密码
2LimitBlankPasswordUse = 0允许空密码登录Windows 默认禁止空密码本地登录，必须手动关闭
3AutoAdminLogon = 1开机自动进桌面跳过登录界面，直接进入 Lab-Admin 桌面
4NoLockScreen = 1禁用锁屏画面避免唤醒后需要滑动/点击才能进入
5DisableCAD = 1禁用 Ctrl+Alt+Del取消安全登录屏幕，直接进桌面
6DisableAutomaticRestartSignOn = 1禁用自动重启登录Windows Update 重启后不弹 Hello 设置向导

安全边界说明

不靠工作站密码保护靠什么保护
物理访问控制实验室门禁 + YubiKey 认证器领取流程
网络安全VLAN 隔离用户网段与管理网段，用户无法触及 NAS 管理口
数据安全NAS 有独立权限控制，用户仅能访问自己的 WS 目录
SaaS 服务安全各服务通过 YubiKey / SSO 独立认证
系统完整性每日自动回滚到基线，无法持久化任何恶意操作

⚠️ 故障排除：重启后仍弹出密码/PIN 界面

如果执行上述命令后重启仍要求输入密码，通常是 Windows Hello PIN 优先级高于密码导致。执行以下命令彻底清除：

REM === 7. 禁用 Windows Hello / PIN 登录 === reg add "HKLM\SOFTWARE\Microsoft\PolicyManager\default\Settings\AllowSignInOptions" /v value /t REG_DWORD /d 0 /f reg add "HKLM\SOFTWARE\Policies\Microsoft\PassportForWork" /v Enabled /t REG_DWORD /d 0 /f REM === 8. 删除已有的 PIN 数据 === takeown /f "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc" /r /d y icacls "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc" /grant Administrators:F /t rd /s /q "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc" REM === 9. 强制登录类型为传统密码（绕过 Hello） === reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v LogonType /t REG_DWORD /d 0 /f REM === 重启验证 === Restart-Computer

备用方案：使用微软官方 Autologon 工具

如果注册表方法仍然无效，可使用 Sysinternals 的 Autologon 工具（微软官方出品）：

# PowerShell 下载并运行 Autologon Invoke-WebRequest -Uri "https://live.sysinternals.com/Autologon.exe" -OutFile "$env:TEMP\Autologon.exe" & "$env:TEMP\Autologon.exe" "I-Lab-003" "" "" # 弹窗后点击 Enable → 重启即可 # 参数说明：Autologon.exe "用户名" "域(留空)" "密码(留空)"

🔍 诊断脚本：一键检查所有状态

如果配置后仍无法自动登录，运行以下脚本查看各项设置当前值，定位问题所在：

REM ============================================ REM 无密码自动登录 - 状态诊断脚本 REM 以管理员身份在 CMD 中运行 REM ============================================ echo. echo ====== 1. 当前用户名 ====== whoami echo. echo ====== 2. 账户密码状态 ====== net user %USERNAME% echo. echo ====== 3. 自动登录注册表 ====== echo [AutoAdminLogon] 期望值: 1 reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon 2>nul echo [DefaultUserName] 期望值: 你的用户名 reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUserName 2>nul echo [DefaultPassword] 期望值: (空) reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultPassword 2>nul echo. echo ====== 4. 空密码策略 ====== echo [LimitBlankPasswordUse] 期望值: 0 reg query "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v LimitBlankPasswordUse 2>nul echo. echo ====== 5. Windows Hello 策略 ====== echo [PassportForWork\Enabled] 期望值: 0 reg query "HKLM\SOFTWARE\Policies\Microsoft\PassportForWork" /v Enabled 2>nul || echo (未设置 - 可能仍启用) echo [AllowSignInOptions] 期望值: 0 reg query "HKLM\SOFTWARE\Microsoft\PolicyManager\default\Settings\AllowSignInOptions" /v value 2>nul || echo (未设置) echo. echo ====== 6. PIN 数据目录 ====== if exist "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc" ( echo [警告] Ngc 目录仍存在! PIN 未彻底清除 dir "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc" /a ) else ( echo [OK] Ngc 目录已删除 ) echo. echo ====== 7. 锁屏与 CAD 策略 ====== echo [NoLockScreen] 期望值: 1 reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\Personalization" /v NoLockScreen 2>nul || echo (未设置 - 锁屏可能仍启用) echo [DisableCAD] 期望值: 1 reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v DisableCAD 2>nul echo. echo ====== 8. PasswordLess 设备策略 (Win11 关键!) ====== echo [DevicePasswordLessBuildVersion] 期望值: 0 (如果为2则会强制要密码!) reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\PasswordLess\Device" /v DevicePasswordLessBuildVersion 2>nul || echo (未找到) echo. echo ====== 9. 电源唤醒密码策略 ====== powercfg /query SCHEME_CURRENT SUB_NONE CONSOLELOCK 2>nul | findstr "Setting" echo. echo ====== 诊断完成 ====== echo 如有 [警告] 项，请按手册对应步骤修复后重启。 pause

常见隐藏原因（按排查优先级）

优先级检查项期望值修复命令
🔴 1 DevicePasswordLessBuildVersion 0 reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\PasswordLess\Device" /v DevicePasswordLessBuildVersion /t REG_DWORD /d 0 /f
🔴 2 Ngc 目录（PIN 数据） 不存在 rd /s /q "C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc"
🟡 3 LimitBlankPasswordUse 0 reg add "HKLM\SYSTEM\CurrentControlSet\Control\Lsa" /v LimitBlankPasswordUse /t REG_DWORD /d 0 /f
🟡 4 AutoAdminLogon 1 reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon /t REG_SZ /d "1" /f

这样所有克隆的机器都会自动具备无密码自动登录能力，无需逐台设置。

2.8 重启 & 验证

# 重启使所有设置生效 Restart-Computer -Force

重启后验证清单

  • 开机后自动登录到 Lab-Admin 桌面（无需点击/输入）
  • 计算机名显示为 GMK-WS-TEMPLATE（Settings → System → About）
  • 时区显示 UTC+8 北京时间
  • 电源计划：不会自动休眠（Settings → System → Power）
  • 远程桌面可从其他设备连入
  • 网卡 WoL 已开启（Device Manager 确认）
  • 文件扩展名已显示
  • 无锁屏画面出现
✅
