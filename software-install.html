安装策略：使用 Windows 自带的 winget 包管理器批量安装， 一条命令搞定。比逐个下载安装包快 10 倍。

3.2 winget 批量安装（推荐）

打开 PowerShell (Admin)，逐块复制粘贴执行：

开发工具

# VS Code - 代码编辑器 winget install -e --id Microsoft.VisualStudioCode --accept-source-agreements --accept-package-agreements # Python 3.13（稳定版，生态兼容性最佳） winget install -e --id Python.Python.3.13 # Git for Windows winget install -e --id Git.Git # Node.js LTS winget install -e --id OpenJS.NodeJS.LTS # Miniconda（Python 虚拟环境管理） winget install -e --id Anaconda.Miniconda3

浏览器

# Google Chrome winget install -e --id Google.Chrome

系统工具

# 7-Zip 压缩工具 winget install -e --id 7zip.7zip # Notepad++ 文本编辑器 winget install -e --id Notepad++.Notepad++ # PuTTY（SSH + 串口终端） winget install -e --id PuTTY.PuTTY # Wireshark（网络分析） winget install -e --id WiresharkFoundation.Wireshark

嵌入式 & IoT

# Arduino IDE 2.x winget install -e --id ArduinoSA.IDE.stable

运行时 & 依赖库

# Visual C++ 2015-2022 Redistributable（大量软件依赖） winget install -e --id Microsoft.VCRedist.2015+.x64 # .NET 8 Runtime（运行 .NET 应用） winget install -e --id Microsoft.DotNet.Runtime.8 # .NET 8 Desktop Runtime（运行 WPF/WinForms 桌面应用） winget install -e --id Microsoft.DotNet.DesktopRuntime.8

远程管理

# RustDesk（开源远程桌面，管理员远程维护用） winget install -e --id RustDesk.RustDesk
⚠️
注意：winget 安装过程中可能会弹出 UAC 提示，点"是"继续。 部分软件安装后需要重新打开 PowerShell 才能在 PATH 中找到。

3.3 一键安装脚本（保存到 NAS 供复用）

将所有安装命令合并为一个脚本，未来重建 Golden Image 时直接运行：

# 保存为 \\192.168.0.81\Software\Scripts\install-lab-tools.ps1 # 或保存到本地 C:\Scripts\install-lab-tools.ps1 $tools = @( "Microsoft.VisualStudioCode", "Python.Python.3.13", "Git.Git", "OpenJS.NodeJS.LTS", "Anaconda.Miniconda3", "Google.Chrome", "7zip.7zip", "Notepad++.Notepad++", "PuTTY.PuTTY", "WiresharkFoundation.Wireshark", "ArduinoSA.IDE.stable", "Microsoft.VCRedist.2015+.x64", "Microsoft.DotNet.Runtime.8", "Microsoft.DotNet.DesktopRuntime.8", "RustDesk.RustDesk" ) Write-Host "=== I-Lab Software Installation ===" -ForegroundColor Cyan Write-Host "Total packages: $($tools.Count)" -ForegroundColor Cyan Write-Host "" $success = 0 $failed = @() foreach ($tool in $tools) { Write-Host "[Installing] $tool ..." -ForegroundColor Yellow winget install -e --id $tool --accept-source-agreements --accept-package-agreements --silent 2>&1 | Out-Null if ($LASTEXITCODE -eq 0 -or $LASTEXITCODE -eq -1978335189) { Write-Host " [OK] $tool" -ForegroundColor Green $success++ } else { Write-Host " [FAIL] $tool (exit: $LASTEXITCODE)" -ForegroundColor Red $failed += $tool } } Write-Host "" Write-Host "=== Results ===" -ForegroundColor Cyan Write-Host "Success: $success / $($tools.Count)" -ForegroundColor Green if ($failed.Count -gt 0) { Write-Host "Failed:" -ForegroundColor Red $failed | ForEach-Object { Write-Host " - $_" -ForegroundColor Red } } Write-Host "" Write-Host "Done! Please restart PowerShell to update PATH." -ForegroundColor Yellow

3.4 VS Code 扩展预配置

安装 VS Code 后，重新打开 PowerShell（让 PATH 生效），然后安装扩展：

# 关闭并重新打开 PowerShell (Admin)，然后执行： # Python 支持 code --install-extension ms-python.python code --install-extension ms-python.vscode-pylance # C/C++（嵌入式开发） code --install-extension ms-vscode.cpptools # 中文语言包 code --install-extension ms-ceintl.vscode-language-pack-zh-hans # 代码格式化 code --install-extension esbenp.prettier-vscode # 远程 SSH（连接其他设备） code --install-extension ms-vscode-remote.remote-ssh # GitHub Copilot（AI 辅助编程） code --install-extension GitHub.copilot # Arduino 支持 code --install-extension vsciot-vscode.vscode-arduino # Jupyter Notebook code --install-extension ms-toolsai.jupyter

3.5 Python 环境验证 & 预装库

# 验证 Python 安装 python --version # 预期输出：Python 3.13.x pip --version # 预期输出：pip 24.x from ... # 升级 pip python -m pip install --upgrade pip # 预装常用数据科学库 pip install numpy pandas matplotlib jupyter scikit-learn # 预装 AI/LLM 相关库（可选，增加 ~3GB 镜像大小） # pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu # pip install transformers openai langchain # 验证 python -c "import numpy; import pandas; print('NumPy:', numpy.__version__); print('Pandas:', pandas.__version__)"
💡
关于 PyTorch + AMD GPU：
AMD Strix Halo 的 ROCm 支持还在早期阶段。如果需要 GPU 加速推理， 建议先用 CPU 版本的 PyTorch，后续 AMD 官方更新 ROCm 驱动后再升级。

3.6 安装 Synology Active Backup Agent

这是 NAS 整机备份的客户端，必须在 Golden Image 中预装。

# 方法一：浏览器下载 # 1. 打开 Chrome，访问: https://192.168.0.81:5001 # 2. 登录 NAS DSM 管理界面 # 3. 打开 Active Backup for Business 套件 # 4. 点击 "PC" 选项卡 → "Add Device" → 下载 Windows Agent # 5. 运行安装程序 # 方法二：如果 Agent 已放在 NAS 共享目录 # 从 NAS 复制安装包到本地运行： Copy-Item "\\192.168.0.81\Software\Agents\Synology Active Backup for Business Agent-x64.msi" -Destination "$env:TEMP\" Start-Process msiexec.exe -ArgumentList "/i `"$env:TEMP\Synology Active Backup for Business Agent-x64.msi`" /quiet" -Wait

Agent 安装后配置

# Agent 安装完后会弹出配置窗口： # 1. Server address: 192.168.0.81 # 2. Username: 你的 NAS 管理员账号 # 3. Password: NAS 密码 # 4. 点 Connect # 5. 成功后在 NAS → Active Backup 界面能看到此设备 # # ⚠️ 注意：Sysprep 后此配置会保留， # 但克隆到新机器后 Agent 会以新 SID 重新注册。 # NAS 上可能出现重复设备，手动删除旧的即可。

3.7 其他手动安装项（不在 winget 中的）

软件安装方式备注
CH340/CH341 USB 转串口驱动 从 NAS \\...\Software\Drivers\CH341\ 手动安装 Arduino Nano/ESP32 等开发板需要
CP2102 USB 转串口驱动 Silicon Labs 官网下载 NodeMCU 等设备需要
FTDI 驱动 FTDI 官网下载 部分高端开发板需要
# 串口驱动安装后验证： # 1. 插入 Arduino/ESP32 到 USB # 2. Device Manager → Ports (COM & LPT) # 3. 应出现 "USB-SERIAL CH340 (COM3)" 或类似 # 4. Arduino IDE 中选择对应 COM 端口

3.8 安装验证

# 重新打开 PowerShell，运行验证脚本： $checks = @( @{Name="Python"; Cmd="python --version"}, @{Name="pip"; Cmd="pip --version"}, @{Name="Git"; Cmd="git --version"}, @{Name="Node.js"; Cmd="node --version"}, @{Name="npm"; Cmd="npm --version"}, @{Name="VS Code"; Cmd="code --version"}, @{Name="conda"; Cmd="conda --version"} ) Write-Host "=== Software Verification ===" -ForegroundColor Cyan foreach ($check in $checks) { Write-Host "$($check.Name): " -NoNewline try { $result = Invoke-Expression $check.Cmd 2>&1 | Select-Object -First 1 Write-Host $result -ForegroundColor Green } catch { Write-Host "NOT FOUND" -ForegroundColor Red } } # 检查 GUI 应用 Write-Host "" Write-Host "=== GUI Apps (check manually) ===" -ForegroundColor Cyan $guiApps = @("Chrome", "Notepad++", "7-Zip", "Wireshark", "Arduino IDE", "RustDesk", "PuTTY") $guiApps | ForEach-Object { Write-Host " ☐ $_" }
  • python/pip/git/node/npm/code 命令行全部可用
  • VS Code 可正常打开，扩展已安装
  • Chrome 可正常上网
  • Arduino IDE 可打开（暂不需要连接设备）
  • Active Backup Agent 显示已连接 NAS
  • RustDesk 可打开并显示 ID
  • NumPy/Pandas 等 Python 库 import 无报错
